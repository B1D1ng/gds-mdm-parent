# Project Glossary

## GDS-MDM-Parent Terminology

This document provides definitions for project-specific terminology and abbreviations used throughout the codebase.

## Core Terminology

| Term | Definition |
|------|------------|
| **GDS** | Golden Data Sets - a data platform capable of supporting automated data set definition, production, processing and maintenance for all CDT Data & Metrics use cases |
| **MDM** | Master Data Management - A method of enabling an organization to link all of its critical data to a common point of reference |
| **Touchpoint** | A user point of interaction between a user and the business |
| **Signal** | An event produced by stateful event processor, based on one or more events |
| **UDF** | User-Defined Function - Custom functions that extend the system's capabilities |
| **DEC** | Data enrichment & consumption component of GDS platform |
| **ESP** | Ebay Signals Platform - Data production component of the GDS platform |

## Technical Abbreviations

| Abbreviation | Full Form |
|--------------|-----------|
| **IT** | Integration Test - Used in test configuration files (application-IT.yaml) |
| **API** | Application Programming Interface |
| **REST** | Representational State Transfer - Architectural style for distributed systems |

## Environment Terminology

| Environment | Description |
|-------------|-------------|
| **Dev** | Development environment for active development work |
| **Feature** | Environment for testing new features in isolation |
| **Staging** | Pre-production environment for integration testing |
| **Pre-Production** | Final testing environment before production deployment |
| **Production** | Live environment serving real users |

## Additional Notes

This glossary is based on the observed file structure and naming conventions. The actual meaning of terms may vary based on the project's specific context. For more detailed information about the project structure and architecture, refer to the other documentation files in this directory.
